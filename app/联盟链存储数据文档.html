fabric 联盟链项目

基本构想

联盟链的发展

  1. 可溯源不可篡改的分布式账本
  2. 同一channel里数据的有偿或无偿共享
  3. 完善合理的权限认证系统（用户做主人，我们做平台）
  4. fabric平台对元数据的再加工
  5. 随着ORG不断加入...各种数据互通有无...
  6. ...不敢想象了

fabric 链上一切皆对象， 我们将要构建一个对象集合世界，一个实物、虚拟物、项目、自然人等等皆可称为对象。用诸多对象去描绘一个概念，比如 描述一个自然人的购买能力可能需要其基本信息（object1）、银行流水（object2）、交易单据（object3）...

现阶段我们只有元数据（metadata），元数据以对象的形式存储在联盟链里，对于新加入链里的组织机构， 它需要贡献它的元数据，之后也将有权使用这些元数据，初步实现一个数据交换共享的策略。这便是我们这个项目的第一步----可溯源不可篡改的分布式账本，也就是接下来要做的事

对象

“一切皆对象” 的理念是产品小哥给出的，其分为两大类：基本对象和行为对象！我转念一想 这两对象足以描绘这宏观世界了。可...正因为这庞杂无章的世界可用那两个对象来描绘，可见其本身有多么的庞大和复杂，尤其是后者。

API 接口

业务无关的基本接口包括：链上对象的增、删、改、查、溯源历史等5个接口。
